Next主题简介

​ Hexo NexT主题是一款非常受欢迎、功能强大的Hexo主题，由Next.Mistaken大神开发。该主题具有响应式设计、多种布局、多种配色方案、多种插件等特点，使得它非常适合用于个人博客、技术博客、科技博客等各类博客网站。同时，Hexo NexT主题还支持多种第三方评论系统、社交分享、代码高亮等功能，为用户提供了更好的使用体验。因此，该主题受到了广大用户的喜爱和追捧。

以下整理均源自：NexT - Theme for Hexo (theme-next.js.org)

安装Next

下载 NexT 有两种推荐的方法任选其一即可。
首先在命令行中将 dir 更改为博客根目录，然后继续执行以下步骤。

  1. NPM
    npm install hexo-theme-next@latest
  2. Git
    git clone https://github.com/next-theme/hexo-theme-next themes/next

Next配置

  1. 建议将node_modules/hexo-theme-next/_config.yml 文件复制到站点根目录并重命名为_config.next.yml
    原因：通过 git 或 npm 升级 NexT 主题时，配置文件会被覆盖，这给主题的配置带来不便，但是此方法需确保您使用的是 Hexo 5.0 或更高版本

  2. 启用Next
    修改站点目录下的 _config.yml(以下均是)文件，将 theme: landscape 修改为 theme: next

  3. 设置缓存支持
    NexT v6+允许缓存内容生成。将 NexT 配置文件(站点根目录下的 _config.next.yml)中的值设置为 true

  4. 方案选择
    方案是NexT支持的一项功能，通过使用Scheme，NexT为您提供不同的视图。几乎所有的配置都可以被这些方案使用。到目前为止，NexT支持4种方案，它们是：

    • Muse→默认方案，这是NexT的初始版本。使用黑白色调，主要看起来干净。
    • Mist→ Muse的紧凑版本，具有整洁的单列视图。
    • Pisces→双列方案，像邻居的女儿一样新鲜。
    • Gemini→ 看起来像双鱼座，但具有带有阴影的不同列块，以显示对视图更敏感。

  5. 配置菜单项
    菜单设置项的格式包含 3 个值：Key: /link/ || icon

    Values Notes
    Key 菜单名称(区分大小写)
    link 网站对应的文件夹
    icon 图标

    每个子菜单项都需要一个default页面。

    默认情况下，NexT 显示不带徽章的菜单项的图标，如需要请做如下配置。

  6. 侧边栏样式
    默认情况下，侧边栏仅显示在帖子中（具有目录），并放置在左侧。您可以通过编辑 NexT 配置文件中设置下的值来更改它。sidebar
    position:设置侧边栏位置，left-左侧，right-右侧
    width:通过以像素为单位指定设置来更改侧边栏宽度，width: 300
    display:具体什么时候显示侧边栏

    • post→具有索引的帖子中显示侧边栏。
    • always→ 在所有页面中显示侧边栏。
    • hide→ 在所有页面中隐藏它（但可以由用户手动打开）。
    • remove→ 完全删除侧边栏。

    padding:通过更改以下值来设置侧边栏填充（以像素为单位），padding: 18
    offset:通过更改以下值来设置侧边栏与顶部菜单栏的偏移量（以像素为单位），offset: 12

  7. 配置头像
    默认情况下，NexT 不会在侧边栏中显示头像。可以通过编辑 NexT 配置文件中avatar的值来配置它。

  8. 其他站点
    默认情况下，NexT 在侧边栏中显示帖子/类别/标签的类别和计数。您可以通过编辑 NexT 配置文件中的值来配置它。site_state

  9. 侧边栏社交链接
    默认情况下，NexT在侧边栏中显示社交链接的图标。

  10. 页脚图标
    默认情况下，NexT 在页脚中显示带有年份和版权信息之间的动画的红色图标。您可以通过编辑 NexT 配置文件部分中的值来配置它。

  11. 发布字数统计
    安装插件：在站点根目录目录中执行以下命令进行安装
    npm install hexo-word-counter
    激活：修改Hexo配置文件(_config.yml)
